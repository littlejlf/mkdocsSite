    页面的生命周期方法

    ChatGPT:

    在微信小程序中，每个页面都有一组生命周期方法，这些方法可以帮助你在页面的不同阶段执行特定的逻辑。以下是页面的主要生命周期方法：

    1. onLoad(options)

      • 页面加载时触发。参数为打开当前页面路径中的参数。
      • 通常用于初始化页面数据。
      onLoad: function(options) {
  console.log('Page onLoad');
  // 初始化页面数据
}

    2. onShow()

      • 页面显示/切入前台时触发。
      • 每次页面显示时都会调用，可以用于页面的刷新。
      onShow: function() {
  console.log('Page onShow');
  // 页面显示时的逻辑
}

    3. onReady()

      • 页面初次渲染完成时触发。
      • 只会调用一次，表示页面已经准备好可以与用户交互。
      onReady: function() {
  console.log('Page onReady');
  // 页面初次渲染完成时的逻辑
}

    4. onHide()

      • 页面隐藏/切入后台时触发。
      • 比如使用 wx.navigateTo 或 wx.redirectTo 跳转到其他页面时会触发。
      onHide: function() {
  console.log('Page onHide');
  // 页面隐藏时的逻辑
}

    5. onUnload()

      • 页面卸载时触发。
      • 比如使用 wx.navigateBack 返回到上一个页面时会触发。
      onUnload: function() {
  console.log('Page onUnload');
  // 页面卸载时的逻辑
}

    6. onPullDownRefresh()

      • 用户下拉动作时触发。
      • 需要在 app.json 中开启 enablePullDownRefresh。
      onPullDownRefresh: function() {
  console.log('Page onPullDownRefresh');
  // 下拉刷新时的逻辑
  wx.stopPullDownRefresh(); // 停止下拉刷新动作
}

    7. onReachBottom()

      • 页面上拉触底事件的处理函数。
      • 通常用于上拉加载更多数据。
      onReachBottom: function() {
  console.log('Page onReachBottom');
  // 上拉加载更多数据的逻辑
}

    8. onShareAppMessage()

      • 用户点击右上角分享时触发。
      • 需要返回一个对象，描述分享内容。
      onShareAppMessage: function() {
  return {
    title: '小程序示例',
    path: '/pages/index/index'
  };
}

    9. onPageScroll(event)

      • 页面滚动时触发。
      • 可以获取到页面滚动的详细信息。
      onPageScroll: function(event) {
  console.log('Page onPageScroll', event.scrollTop);
  // 页面滚动时的逻辑
}

    10. onResize(resize)

      • 页面尺寸变化时触发。
      • 用于处理横竖屏切换等场景。
      onResize: function(resize) {
  console.log('Page onResize', resize.size);
  // 页面尺寸变化时的逻辑
}

    11. onTabItemTap(item)

      • 当前是 tab 页时，点击 tab 时触发。
      • 可以获取到被点击的 tab 的详细信息。
      onTabItemTap: function(item) {
  console.log('Page onTabItemTap', item);
  // 点击 tab 时的逻辑
}

    通过这些生命周期方法，你可以更好地控制页面的行为和状态，确保用户在不同阶段的体验是一致的。
